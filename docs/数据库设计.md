# 智能评审 — 数据库设计（草案）

本文在前端类型（`MockRule`、`RiskItem` 等）与《规则详情页-需求说明》基础上，整理**推荐的关系型库表结构**，便于与 BPM、大模型任务、审核页数据对齐。  
**枚举与字段语义**须与审核界面（Reviewer）保持一致。

---

## 1. 实体关系（概览）

```
组织/公司 ──┬── 部门
            │
规则(rule) ─┼── 规则版本(rule_version)     （1 : N，版本不可改，仅追加）
            │
            └── 规则字段模板(rule_field_template) （1 : N，顺序可控）

审核场景(review_scenario) ── 场景规则(review_scenario_rule) ── 规则(rule)  （N : M，带排序）

合同(contract) ── 合同段落(contract_paragraph)  （解析产物，供 locationId）

BPM 审核任务(review_task) ── 合同 + 场景快照 + 状态
         │
         ├── 模型运行(ai_inference_run)           （可选：一次或多次调用）
         │
         └── 评审结果项(review_finding) ── 对应规则版本快照、可映射 RiskItem

评审任务用户态(review_task_user_state)          （审核意见、展开态等，按任务+用户）
```

---

## 2. 枚举建议（CHECK 或字典表）

| 枚举 | 取值 | 说明 |
|------|------|------|
| `rule_status` | `draft`, `active`, `inactive` | 草稿 / 已生效 / 已停用；仅停用后可删规则（产品约束） |
| `rule_output_mode` | `bpm_structured`, `recognition_only` | 与前端 `RuleOutputMode` / `RuleMode` 一致 |
| `risk_category` | `trigger`, `missing`, `low` | 模型对单条结果的判定，非规则配置字段 |
| `scenario_status` | `active`, `inactive` | 场景启用 |

---

## 3. 表结构说明

### 3.1 `rule`（规则主表，当前生效指针）

| 列名 | 类型 | 约束 | 说明 |
|------|------|------|------|
| `id` | UUID / BIGINT | PK | 规则稳定 ID（如 R001 可映射为业务编码字段） |
| `code` | VARCHAR(64) | UNIQUE 可空 | 对外展示编码，可选 |
| `name` | VARCHAR(256) | NOT NULL | 对应 `RiskItem.title` 来源 |
| `module` | VARCHAR(128) | NOT NULL | 所属模块/组织维度 |
| `description` | TEXT | | 规则说明 |
| `status` | rule_status | NOT NULL | |
| `output_mode` | rule_output_mode | NOT NULL | 评审结果 vs 仅识别 |
| `current_version_id` | UUID / BIGINT | FK → rule_version.id | 当前发布版本 |
| `created_at` / `updated_at` | TIMESTAMPTZ | | |

> **提示词不落主表**：以版本表为准，避免与「版本历史」双写不一致。

---

### 3.2 `rule_version`（规则版本，不可变快照）

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | PK | |
| `rule_id` | FK → rule.id | |
| `version_no` | INT | 单调递增，展示用 v3 |
| `system_prompt` | TEXT | 系统提示词快照 |
| `user_prompt` | TEXT | 用户提示词快照（含 `{{变量}}` 原文） |
| `change_note` | VARCHAR(512) | 变更摘要 |
| `created_by` | FK → user.id | |
| `created_at` | TIMESTAMPTZ | |

**索引**：`(rule_id, version_no)` UNIQUE。

> **回滚**：回滚 = 复制某历史版本内容插入新版本（新版本号），不覆盖旧行（与需求文档一致）。

---

### 3.3 `rule_field_template`（需提取字段模板）

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | PK | |
| `rule_version_id` | FK → rule_version.id | 绑定到**具体版本**，保证重放一致 |
| `field_name` | VARCHAR(128) | 对应 `StructuredField.label` 的模板名 |
| `sort_order` | INT | 展示与模型提示中的顺序 |

**索引**：`(rule_version_id, sort_order)`。

> `output_mode = recognition_only` 时可为空表。

### 3.10 `review_finding`（一条评审结果 ≈ `RiskItem`）

与前端 `RiskItem` 对齐；**增加规则外键与版本快照**。

| 列名 | 类型 | 说明 |
|------|------|------|
| `id` | PK | 对应前端 `risk-1` 类 ID 可映射为 UUID |
| `review_task_id` | FK | |
| `inference_run_id` | FK 可空 | |
| `rule_id` | FK | **建议必填**，用于追溯 |
| `rule_version_id` | FK | 生成该条时使用的规则版本 |
| `output_mode` | rule_output_mode | 冗余自规则版本，便于查询 |
| `category` | risk_category | 模型输出 |
| `title` | VARCHAR(256) | 可与规则名一致或模型生成 |
| `summary` / `details` / `conclusion` / `reason` | TEXT | |
| `quote` | TEXT | 可空 |
| `location_id` / `reason_location_id` | VARCHAR(128) | 关联 `contract_paragraph.paragraph_key` |
| `sort_order` | INT | 与同任务内多条结果的展示顺序 |
| `created_at` | | |

**用户操作态**（二选一）：

- **方案 A**：本表加 `is_accepted`, `accepted_after_edit`, `ignored`, `recognition_feedback` 等（与现前端一致）。
- **方案 B**：拆 `review_finding_user_state`（任务+用户+finding 维度），适合多人会签。

结构化子表见下。

## 4. 与大模型任务的衔接（数据流）

1. 根据 `review_task` → `contract` + `contract_paragraph` 组装带 **paragraph_key** 的正文。  
2. 根据 `scenario` 解析出规则列表 → 取各规则 **`current_version_id`**（或任务快照中的版本）→ 读 `system_prompt`、`user_prompt`、`rule_field_template`。  
3. 服务端替换用户提示词变量（`{{规则名}}` 等），拼接字段模板说明，调用模型。  
4. 解析 JSON → 写入 `review_finding` + 子表；`category` / `structuredFields` / `recognitionClauses` 与表 §3.10–3.12 对应。  
5. 审核页加载 `review_task` 下 findings，写回用户态字段或 `review_task_user_state`。

---

## 5. 与当前前端 Mock 的差异

| 点 | 说明 |
|----|------|
| 规则与结果 | 前端 `RiskItem` **无 `rule_id`**；库表建议 **必须有**，便于 BPM 回传与统计。 |
| 规则配置 | `mockRules` 与 `riskItems` **未打通**；落库后以 `review_finding.rule_version_id` 为权威快照。 |
| 审核场景 | 需求已定义；库表为 §3.4–3.5，前端菜单可后续补。 |

---

## 6. 索引与约束建议（摘要）

- `review_finding(review_task_id, sort_order)`  
- `review_finding(review_task_id, rule_id)`  
- `contract_paragraph(contract_id, paragraph_key)` UNIQUE  
- 外键删除策略：版本、段落建议 **RESTRICT**；任务完成后的 finding 建议 **禁止物理删** 或软删。
